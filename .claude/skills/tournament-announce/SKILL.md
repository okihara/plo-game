# Tournament Announce

plo-game（BabyPLO）の毎日トナメ **開催告知** ツイート下書きを作るスキル。結果ツイート（`/tournament-tweet`）と対になる、開催前の宣伝用。

## 入力

- 引数なし → 今日の日付で標準的な告知を生成
- 引数あり → 日付・曜日・特記事項（新機能、プレゼント企画、祝日など）をフリーテキストで受け取る
  - 例: `/tournament-announce 金曜はAmazonギフト券1000円`
  - 例: `/tournament-announce 4/20 月曜 AIレビュー機能リリース直後`

## 固定情報

- **トナメ名プレフィックス**: `BabyPLO Daily M/D`（曜日物・祝日はユーザー指示に従って `Holiday` `Blue Monday` `Happy Friday` などに置換）
- **開始時刻**: 22:00（固定）
- **URL**: `https://baby-plo.app`
- **ハッシュタグ**: `#BabyPLO`
- **参加費**: 無料（フリーロール）

日付が未指定なら「今日」「今夜」を優先し、必要な箇所だけ `M/D` を埋める。

### 曜日別バリアント（週次スケジュール）

今夜のトナメ種別は曜日で決まる。告知文に **「今夜は <variant>」** という形で必ず織り込む（パターンA〜Cどれでも自然な位置に1回）。

| 曜日 | バリアント |
|---|---|
| 月曜 | PLO |
| 火曜 | PLO |
| 水曜 | PLO8（Hi-Lo） |
| 木曜 | PLO |
| 金曜 | PLO（優勝者にAmazonギフト券1,000円分） |
| 土曜 | 5-Card PLO |
| 日曜 | PLO Double Boards Bomb Pot |

- 通常PLO日（月・火・木）はバリアントに特別に触れなくてもOK（触れる場合は「通常のPLO」程度）
- 水曜の `PLO8`、土曜の `5-Card`、日曜の `Double Boards Bomb Pot` は**必ず明記**（普段と違うバリアントなので集客の核）
- 金曜は **Amazonギフト券特典**を必ず本文に入れる（バリアントは通常PLO）

## Workflow

### Step 1: 昨夜のトナメ結果を取得

プロジェクトルート（`/Users/masa/work/plo-game`）から以下を実行する。**接続URLをコマンドラインに載せない。** スクリプトは `server/.env` の `DATABASE_PROD_PUBLIC_URL` を自動で読む。

```bash
cd server && npx tsx scripts/tournament-announce-data.ts --prod
```

stdout に JSON が出る。主要フィールド:

- `tournament.name` / `completedAt` / `hoursAgo` / `stale`（48h超なら true）/ `totalEntries` / `uniqueRegistrations`
- `winner.displayName`

**取得結果の扱い:**

- `tournament` が `null` または `stale: true` の場合 → 「昨夜の結果」行は **入れない**（無理に古いデータを使わない）。3案は従来通り生成する
- それ以外 → 3案すべてに「昨夜の結果に触れる1行」を織り込む（Step 3 参照）

### Step 2: 文脈整理

ユーザー入力から以下を抽出する（なければ省略）:

- 日付 / 曜日 / 祝日
- 特典・キャンペーン（例: 金曜の Amazon ギフト券）
- 新機能・直近のアップデート（例: AIレビュー）
- 気分・トーン寄せ（例: 月曜の憂鬱、連休中、平日夜）

### Step 3: ツイート下書きを **3案** 生成

既存サンプルの **3パターン** を踏襲し、それぞれ別トーンで1本ずつ出す。

**【冒頭2行は全パターン共通・必須】**

```
参加無料のオンラインPLOトーナメント
今夜も22:00から開催です！
```

3案すべてこの2行で始める。改変・省略・順序入れ替えは禁止。続く本文だけをパターンごとに書き分ける。

**【昨夜の結果1行】（Step 1 でデータが取れて `stale: false` のとき必須）**

3案それぞれで、表現を変えて織り込む。同じフレーズの使い回しは禁止。語彙の例:

- `昨夜は <winner> さんが <totalEntries>エントリーを制して優勝🏆`
- `昨夜の優勝は <winner> さん（<totalEntries>エントリー）`
- `昨夜は <totalEntries>エントリー、<winner> さんが頂点に🏆`
- `昨夜も<totalEntries>エントリー集まって <winner> さんが優勝！`

各パターンで挿入位置は本文の頭か中ほど、自然な所に1回だけ。優勝者名はマスク済みなので加工しない。

**パターンA: 機能・メリット訴求型**

```
参加無料のオンラインPLOトーナメント
今夜も22:00から開催です！

<昨夜の結果1行>
フリーロールで気軽に参加できて、PLOトナメの練習にもぴったり

<特記事項があればここに1文。例: 今日追加された〇〇機能もあわせて使えば、振り返りまでしっかりできます💪>

#BabyPLO
https://baby-plo.app
```

**パターンB: ノリ・感情訴求型（短め）**

```
参加無料のオンラインPLOトーナメント
今夜も22:00から開催です！

<昨夜の結果1行>
<曜日・気分に寄せたフック1〜2行>
<一言あおり>💪

#BabyPLO
https://baby-plo.app
```

- 例フック: 「PLOで憂鬱な月曜から逃避しましょう！」「休日かどうかなんて関係ない！」「平日の夜こそPLO！」

**パターンC: 実績・特典訴求型**

`連日30エントリー超え` の汎用文を、**昨夜の実数（`totalEntries`）に置き換える**。

```
参加無料のオンラインPLOトーナメント
今夜も22:00から開催です！

昨夜も<totalEntries>エントリー集まりました！（優勝は <winner> さん🏆）
気軽に実戦練習ができるので、<特典があればここ。例: 金曜は優勝者にAmazonギフト券1,000円分をご用意しています！>

練習したい人も、しっかり勝ちを狙いたい人も、ぜひご参加ください！
#BabyPLO
https://baby-plo.app
```

### Step 4: 生成ルール

- **冒頭2行（「参加無料のオンラインPLOトーナメント / 今夜も22:00から開催です！」）は全案で必ず入れる**
- 昨夜の結果1行は **3案すべてで表現を変える**（同じ言い回しを2案以上で使うのは禁止）
- データが取れなかった or `stale: true` のときは、昨夜の結果行を**省略**して従来の汎用文（パターンCなら「連日30エントリー超え」）に戻す
- 絵文字は既存サンプルと同じ `💪` を基本に、増やしすぎない。`🏆` は昨夜の結果1行で使う想定。`🔥` `🎯` は控えめに1個まで
- ハッシュタグは **`#BabyPLO` のみ**（複数付けない）
- URL は末尾 or ハッシュタグの直後に1回だけ
- 特記事項がない日は、AIレビューなど **直近の機能には触れない**（古くなった情報を使い回さない）
- 140字制限は意識するが、告知は長めでも可。既存サンプルも長短混在

### Step 5: 出力

3案それぞれをコードブロックで並べて提示する。各案の前に **どのトーン狙いか**を1行で添える（例: `# パターンA: 機能訴求`）。昨夜の結果を使った場合は最後の一言の前に **どのデータを使ったか**（優勝者名・エントリー数・対象トナメ名）を1行で補足する。最後に「どれをベースに詰めますか？」と一言だけ確認する。

ファイル保存・メモリ保存は **しない**（毎日書き捨て）。
