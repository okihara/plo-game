# Tournament Announce

plo-game（BabyPLO）の毎日トナメ **開催告知** ツイート下書きを作るスキル。結果ツイート（`/tournament-tweet`）と対になる、開催前の宣伝用。

## 入力

- 引数なし → 今日の日付で標準的な告知を生成
- 引数あり → 日付・曜日・特記事項（新機能、プレゼント企画、祝日など）をフリーテキストで受け取る
  - 例: `/tournament-announce 金曜はAmazonギフト券1000円`
  - 例: `/tournament-announce 4/20 月曜 AIレビュー機能リリース直後`

## データ取得

**不要。** DB参照もスクリプト実行もしない。ユーザーから渡された文脈と、下記の固定情報だけを使う。

- **トナメ名プレフィックス**: `BabyPLO Daily M/D`（曜日物・祝日はユーザー指示に従って `Holiday` `Blue Monday` `Happy Friday` などに置換）
- **開始時刻**: 22:00（固定）
- **URL**: `https://babyplo.app`
- **ハッシュタグ**: `#BabyPLO`
- **参加費**: 無料（フリーロール）

日付が未指定なら「今日」「今夜」を優先し、必要な箇所だけ `M/D` を埋める。

## Workflow

### Step 1: 文脈整理

ユーザー入力から以下を抽出する（なければ省略）:

- 日付 / 曜日 / 祝日
- 特典・キャンペーン（例: 金曜の Amazon ギフト券）
- 新機能・直近のアップデート（例: AIレビュー）
- 気分・トーン寄せ（例: 月曜の憂鬱、連休中、平日夜）

### Step 2: ツイート下書きを **3案** 生成

既存サンプルの **3パターン** を踏襲し、それぞれ別トーンで1本ずつ出す。

**【冒頭2行は全パターン共通・必須】**

```
参加無料のオンラインPLOトーナメント
今夜も22:00から開催です！
```

3案すべてこの2行で始める。改変・省略・順序入れ替えは禁止。続く本文だけをパターンごとに書き分ける。

**パターンA: 機能・メリット訴求型**

```
参加無料のオンラインPLOトーナメント
今夜も22:00から開催です！

フリーロールで気軽に参加できて、PLOトナメの練習にもぴったり

<特記事項があればここに1文。例: 今日追加された〇〇機能もあわせて使えば、振り返りまでしっかりできます💪>

#BabyPLO
https://babyplo.app
```

**パターンB: ノリ・感情訴求型（短め）**

```
参加無料のオンラインPLOトーナメント
今夜も22:00から開催です！

<曜日・気分に寄せたフック1〜2行>
<一言あおり>💪

#BabyPLO
https://babyplo.app
```

- 例フック: 「PLOで憂鬱な月曜から逃避しましょう！」「休日かどうかなんて関係ない！」「平日の夜こそPLO！」

**パターンC: 実績・特典訴求型**

```
参加無料のオンラインPLOトーナメント
今夜も22:00から開催です！

連日30エントリー超え、気軽に実戦練習ができます！
<特典があればここ。例: 金曜は優勝者にAmazonギフト券1,000円分をご用意しています！>

練習したい人も、しっかり勝ちを狙いたい人も、ぜひご参加ください！
#BabyPLO
https://babyplo.app
```

### Step 3: 生成ルール

- **冒頭2行（「参加無料のオンラインPLOトーナメント / 今夜も22:00から開催です！」）は全案で必ず入れる**
- 絵文字は既存サンプルと同じ `💪` を基本に、増やしすぎない。`🔥` `🎯` は控えめに1個まで
- ハッシュタグは **`#BabyPLO` のみ**（複数付けない）
- URL は末尾 or ハッシュタグの直後に1回だけ
- 特記事項がない日は、AIレビューなど **直近の機能には触れない**（古くなった情報を使い回さない）
- 140字制限は意識するが、告知は長めでも可。既存サンプルも長短混在

### Step 4: 出力

3案それぞれをコードブロックで並べて提示する。各案の前に **どのトーン狙いか**を1行で添える（例: `# パターンA: 機能訴求`）。最後に「どれをベースに詰めますか？」と一言だけ確認する。

ファイル保存・メモリ保存は **しない**（毎日書き捨て）。
